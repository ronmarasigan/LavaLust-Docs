What is OOP?

  • OOP stands for Object-Oriented Programming.
  • Procedural programming focuses on functions that operate on data.
  • Object-oriented programming focuses on objects that bundle data (properties) and functions (methods).

Why Use OOP in PHP?

  • Faster and easier to execute than sprawling procedural codebases.
  • Provides a clear structure for programs.
  • Encourages the DRY principle (Don’t Repeat Yourself) for better reusability.
  • Easier to maintain, modify, and debug.
  • Enables reusable applications with less code and shorter development time.

Tip: The DRY Principle

  • Reduce repetition by extracting common logic into single, reusable classes or methods.
  • Reference that shared code instead of copying the same blocks across your application.

PHP — What are Classes and Objects?

  • Class: A template/blueprint describing properties and methods.
  • Object: An instance of a class created at runtime.
  • Objects inherit the structure (properties/methods) from their class, but can hold different values.

Examples

<?php
// Example 1: Fruit (Class: Fruit; Objects: Apple, Banana, Mango)
class Fruit {
    public $name;
}

$apple = new Fruit();
$apple->name = "Apple";

$banana = new Fruit();
$banana->name = "Banana";

$mango = new Fruit();
$mango->name = "Mango";

echo $apple->name;   // Apple
echo $banana->name;  // Banana
echo $mango->name;   // Mango

<?php
// Example 2: Car (Class: Car; Objects: Volvo, Audi, Toyota)
class Car {
    public $brand;
}

$volvo = new Car();
$volvo->brand = "Volvo";

$audi = new Car();
$audi->brand = "Audi";

$toyota = new Car();
$toyota->brand = "Toyota";

echo $volvo->brand; // Volvo
echo $audi->brand;  // Audi
echo $toyota->brand;// Toyota

Key Points to Remember

  • A class defines properties (data) and methods (functions).
  • An object is created from a class and can store its own values.
  • OOP helps you write modular, reusable, and scalable PHP code.
