🧩 PHP — Introduction to Object-Oriented Programming (OOP)

📘 What is OOP?

Object-Oriented Programming (OOP) is a programming paradigm that models real-world entities as objects. Each object bundles data (properties) and behavior (methods). Starting with PHP 5, PHP provides full support for writing object-oriented code.

⚖️ Procedural vs Object-Oriented

  • Procedural: Write functions and procedures that operate on global or passed data.
  • Object-Oriented: Create classes (templates) and instantiate objects that encapsulate state and behavior.

🛠 Why use OOP in PHP?

  • Clear structure: organizes code into logical components (classes).
  • Reusability: classes can be reused across projects.
  • DRY (Don't Repeat Yourself): reduces duplication by centralizing functionality.
  • Maintainability: easier to modify and debug smaller, focused classes.
  • Extensibility: inheritance and interfaces allow expanding behavior without altering existing code.

📚 Basic OOP Concepts

Class

A class is a blueprint that defines properties and methods.

Object

An object is an instance of a class with its own property values.

Property & Method

Property: a variable that holds the object's state.
Method: a function that defines the object's behavior.

🔧 Simple Example

<?php
class Car {
    public $brand;
    public $year;

    public function __construct($brand, $year) {
        $this->brand = $brand;
        $this->year = $year;
    }

    public function getInfo() {
        return "{$this->brand} ({$this->year})";
    }
}

// Create objects (instances)
$toyota = new Car("Toyota", 2020);
$audi   = new Car("Audi", 2022);

echo $toyota->getInfo(); // Toyota (2020)
echo $audi->getInfo();   // Audi (2022)
?>

🏷 Access Modifiers

Control visibility of properties and methods:

  • public — accessible anywhere.
  • protected — accessible in the class and subclasses.
  • private — accessible only inside the class.

🧭 Encapsulation

Encapsulation means bundling data and methods while hiding internal state. Use getters/setters to control access.

🔁 Inheritance

Inheritance allows a class to extend another class and reuse or override its functionality.

class Vehicle {
    public function start() {
        return "Vehicle started";
    }
}

class Motorcycle extends Vehicle {
    public function start() {
        return "Motorcycle started";
    }
}

$m = new Motorcycle();
echo $m->start(); // Motorcycle started

🔀 Polymorphism

Polymorphism lets different classes be treated through a common interface or base class. Methods with the same name behave differently depending on the object type.

📌 DRY — Don't Repeat Yourself

Extract repeated logic into a single class or method and reuse it across your application. This reduces bugs and keeps code maintainable.

📌 Quick Summary

  • OOP = classes + objects.
  • Use public, protected, private to control visibility.
  • Encapsulation, Inheritance, Polymorphism are core OOP principles.
  • OOP helps create modular, reusable, maintainable PHP applications.
