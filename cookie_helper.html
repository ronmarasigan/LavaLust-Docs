The Cookie Helper provides functions to easily manage cookies in your LavaLust PHP Framework application.

Note: These functions are part of the LavaLust framework and require the framework to be properly initialized.

set_cookie()

Description

Sets a cookie with the specified name and value.

Syntax

set_cookie(string $name, string $value = '', int $expiration = 0, array $options = array())

Parameters

  • $name (string) - The name of the cookie.
  • $value (string) - The value of the cookie. Default: empty string.
  • $expiration (int) - The time the cookie expires. This is a Unix timestamp. Default: 0 (expires when browser closes).
  • $options (array) - Additional cookie options as an associative array. Possible keys:
    • domain - Cookie domain
    • path - Cookie path
    • prefix - Cookie name prefix
    • secure - If TRUE, only transmit over HTTPS
    • httponly - If TRUE, only accessible via HTTP (no JavaScript)

Return Value

void - This function does not return anything.

Example

// Set a cookie that expires in 1 hour
set_cookie('username', 'john_doe', time() + 3600);

// Set a cookie with additional options
set_cookie('prefs', 'dark_mode', time() + 86400, [
    'domain' => '.example.com',
    'path' => '/',
    'secure' => true,
    'httponly' => true
]);

get_cookie()

Description

Retrieves the value of a cookie with the specified name.

Syntax

get_cookie(string $name)

Parameters

  • $name (string) - The name of the cookie to retrieve.

Return Value

mixed - The value of the cookie if it exists, NULL otherwise.

Example

// Get a cookie value
$username = get_cookie('username');

if ($username !== null) {
    echo "Welcome back, " . htmlspecialchars($username);
} else {
    echo "Please log in";
}

delete_cookie()

Description

Deletes a cookie by setting its value to empty and expiration to the past.

Syntax

delete_cookie(string $name, string $domain = '', string $path = '/', string $prefix = '')

Parameters

  • $name (string) - The name of the cookie to delete.
  • $domain (string) - The domain the cookie was set for. Default: empty string.
  • $path (string) - The path the cookie was set for. Default: '/'.
  • $prefix (string) - The prefix used when the cookie was set. Default: empty string.

Return Value

void - This function does not return anything.

Example

// Delete a cookie
delete_cookie('username');

// Delete a cookie with specific domain and path
delete_cookie('prefs', '.example.com', '/account');

Notes

  • These functions internally use the framework's I/O component (lava_instance()->io).
  • Cookie prefixes can be configured in the framework's configuration file.
  • For security, consider always setting the secure and httponly flags when setting cookies.
  • Cookie values are automatically escaped when set and unescaped when retrieved.
