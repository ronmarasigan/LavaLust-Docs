üßÆ Arrays

Definition: Arrays are fixed-size, indexed collections of elements of the same type.

int[] numbers = { 10, 20, 30 };
        Console.WriteLine(numbers[1]); // Output: 20

Limitations: Fixed size, no built-in methods for dynamic resizing or advanced operations.

üìê Types of Arrays

  • One-Dimensional Array: A linear sequence of elements.
    • string[] names = new string[3];
        names[0] = "Alice";
        names[1] = "Bob";
        names[2] = "Charlie";
  • Two-Dimensional Array: A grid or matrix of elements.
    • int[,] matrix = new int[2, 3] {
        {1, 2, 3},
        {4, 5, 6}
        };
        Console.WriteLine(matrix[1, 2]); // Output: 6
  • Jagged Array: An array of arrays, where each sub-array can have a different length.
    • int[][] jagged = new int[2][];
        jagged[0] = new int[] { 1, 2 };
        jagged[1] = new int[] { 3, 4, 5 };
        Console.WriteLine(jagged[1][2]); // Output: 5

üìã List<T>

Definition: A dynamic array that can grow or shrink. Part of System.Collections.Generic.

using System.Collections.Generic;

        List<string> fruits = new List<string>();

        // ‚úÖ Add
        fruits.Add("Apple");
        fruits.Add("Banana");

        // ‚úÖ Access
        Console.WriteLine(fruits[0]); // Output: Apple

        // ‚úÖ Update
        fruits[1] = "Mango"; // Replaces Banana with Mango

        // ‚úÖ Delete
        fruits.Remove("Apple"); // Removes by value
        fruits.RemoveAt(0);     // Removes by index

        // ‚úÖ Check
        Console.WriteLine(fruits.Contains("Mango")); // Output: True

Use Case: When you need a resizable array with built-in methods like Add, Remove, Contains.

üîë Dictionary<TKey, TValue>

Definition: A collection of key-value pairs. Keys must be unique.

Dictionary<string, int> ages = new Dictionary<string, int>();

        // ‚úÖ Add
        ages["Alice"] = 30;
        ages.Add("Bob", 25);

        // ‚úÖ Access
        Console.WriteLine(ages["Bob"]); // Output: 25

        // ‚úÖ Update
        ages["Bob"] = 26; // Updates Bob's age

        // ‚úÖ Delete
        ages.Remove("Alice");

        // ‚úÖ Check
        Console.WriteLine(ages.ContainsKey("Bob")); // Output: True

Use Case: Fast lookup by key, mapping identifiers to values.

üöö Queue<T>

Definition: A first-in, first-out (FIFO) collection.

Queue<string> tasks = new Queue<string>();

        // ‚úÖ Add
        tasks.Enqueue("Task 1");
        tasks.Enqueue("Task 2");

        // ‚úÖ Access (Peek without removing)
        Console.WriteLine(tasks.Peek()); // Output: Task 1

        // ‚úÖ Delete (Dequeue removes front item)
        string completed = tasks.Dequeue(); // Task 1

        // ‚úÖ Check
        Console.WriteLine(tasks.Contains("Task 2")); // Output: True

Use Case: Processing items in the order they were added.

üóÇÔ∏è Stack<T>

Definition: A last-in, first-out (LIFO) collection.

Stack<string> history = new Stack<string>();

        // ‚úÖ Add
        history.Push("Page A");
        history.Push("Page B");

        // ‚úÖ Access (Peek without removing)
        Console.WriteLine(history.Peek()); // Output: Page B

        // ‚úÖ Delete (Pop removes top item)
        string lastPage = history.Pop(); // Page B

        // ‚úÖ Check
        Console.WriteLine(history.Contains("Page A")); // Output: True

Use Case: Undo operations, backtracking, nested parsing.

üß© HashSet<T>

Definition: A collection of unique elements. Fast lookup and no duplicates.

HashSet<int> ids = new HashSet<int>();

        // ‚úÖ Add
        ids.Add(101);
        ids.Add(102);
        ids.Add(101); // Duplicate ignored

        // ‚úÖ Access (Check membership)
        Console.WriteLine(ids.Contains(102)); // Output: True

        // ‚úÖ Delete
        ids.Remove(101);

        // ‚úÖ Count
        Console.WriteLine(ids.Count); // Output: 1

Use Case: Membership testing, enforcing uniqueness.

üìò Summary

  • Arrays are fixed-size and best for simple, static data.
  • List<T> is a dynamic array with rich methods.
  • Dictionary<TKey, TValue> maps keys to values for fast lookup.
  • Queue<T> processes items in FIFO order.
  • Stack<T> processes items in LIFO order.
  • HashSet<T> stores unique elements with fast membership testing.
Educator Tip: Use arrays for fixed-size data and collections for dynamic, flexible structures. Encourage students to choose the right tool based on access patterns, mutability, and performance needs.
