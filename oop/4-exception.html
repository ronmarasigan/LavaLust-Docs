üí• What is an Exception?

Definition: An exception is an unexpected event or error that occurs during program execution, disrupting the normal flow of instructions.

Common Causes:

  • Invalid user input
  • File not found
  • Division by zero
  • Null reference access

Without proper handling, exceptions can crash your program. C# provides structured mechanisms to detect and respond to these errors gracefully.

üõ°Ô∏è try-catch Block

Definition: The try block contains code that might throw an exception. The catch block handles the error if it occurs.

try {
    int x = 10;
    int y = 0;
    int result = x / y; // Division by zero
} catch (DivideByZeroException ex) {
    Console.WriteLine("Error: Cannot divide by zero.");
}

Tip: You can catch specific exceptions or use a general Exception type for broader handling.

üîí finally Block

Definition: The finally block runs regardless of whether an exception occurred. It's ideal for cleanup tasks like closing files or releasing resources.

try {
    Console.WriteLine("Opening file...");
    // Simulate file operation
} catch (Exception ex) {
    Console.WriteLine("Something went wrong.");
} finally {
    Console.WriteLine("Closing file.");
}

üö® throw Statement

Definition: Use throw to manually raise an exception when a specific condition is met.

int age = -5;

if (age < 0) {
    throw new ArgumentOutOfRangeException("Age cannot be negative.");
}

Use Case: Input validation, enforcing business rules, signaling illegal states.

üìò Summary

  • try detects risky code
  • catch handles specific or general exceptions
  • finally ensures cleanup
  • throw lets you raise custom errors
