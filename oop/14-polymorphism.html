Polymorphism & Overriding Methods

Runtime flexibility through virtual, override, and sealed override

üß† Concept

Polymorphism allows objects of different types to be treated through a common base type while executing behavior defined by their actual type.

  • Base references can point to derived objects
  • Behavior is determined by the actual object type
  • Requires virtual in base and override in derived

üîß Virtual Methods

Mark a method virtual in the base class to allow derived classes to override it.

public class Animal
{
    public virtual void Speak() => Console.WriteLine("Generic animal sound");
}
üí° Tip: Virtual methods form the basis of runtime polymorphism. Without them, derived classes cannot override base behavior.

üîÅ Overriding Behavior

Use override in a derived class to provide a specific implementation of a virtual method.

public class Dog : Animal
{
    public override void Speak() => Console.WriteLine("Bark");
}

public class Cat : Animal
{
    public override void Speak() => Console.WriteLine("Meow");
}

When accessed via a base reference, the overridden method executes:

Animal a1 = new Dog();
Animal a2 = new Cat();

a1.Speak();  // Output: Bark
a2.Speak();  // Output: Meow

üõ°Ô∏è Sealed Override

Use sealed override to prevent further overriding of a method in subclasses.

public class Logger
{
    public virtual void Log() => Console.WriteLine("Base log");
}

public class FileLogger : Logger
{
    public sealed override void Log() => Console.WriteLine("File log");
}

public class SecureLogger : FileLogger
{
    // ‚ùå Cannot override Log() again
    // public override void Log() => Console.WriteLine("Secure log");
}
üí° Tip: Sealing overrides enforces design constraints and guards against unintended behavior changes downstream.

üöÄ Runtime Dispatch

Polymorphism enables dynamic method dispatch‚Äîmethod calls are resolved at runtime based on the actual object type.

List<Animal> zoo = new List<Animal> {
    new Animal(),
    new Dog(),
    new Cat()
};

foreach (var creature in zoo)
{
    creature.Speak();  // Output varies: "Generic animal sound", "Bark", "Meow"
}

This pattern is common in frameworks, game engines, and UI systems where objects exhibit type-specific behavior.

üìò Summary

  • virtual marks methods for overriding
  • override customizes behavior in derived classes
  • sealed override prevents further overrides
  • Base references invoke derived implementations
  • Method dispatch is resolved at runtime based on actual object type
