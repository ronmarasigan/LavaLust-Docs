Inheritance in C#

Reuse and extend functionality using base and derived classes

üîç Overview

Inheritance is an object-oriented principle that allows one class to acquire the members of another. It promotes code reuse and logical hierarchy.

  • Base class: the parent class
  • Derived class: the child class that inherits
  • Use the : symbol to inherit

üìê Definition

Inheritance allows a class to extend another class, gaining access to its fields and methods. The derived class can also override or extend behavior.

public class Animal
{
    public void Eat()
    {
        Console.WriteLine("Eating...");
    }
}

public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine("Barking...");
    }
}

Dog inherits from Animal and can use both Eat() and Bark().

üîß Types of Inheritance

  • Single Inheritance: One base class, one derived class
  • Multilevel Inheritance: A class inherits from a derived class
  • Hierarchical Inheritance: Multiple classes inherit from one base class
üí° C# does not support multiple inheritance directly, but interfaces can simulate it.

üíª Examples

1. Single Inheritance

public class Vehicle
{
    public void Start() => Console.WriteLine("Vehicle started.");
}

public class Car : Vehicle
{
    public void Drive() => Console.WriteLine("Car is driving.");
}

// Usage
Car c = new Car();
c.Start();  // Inherited
c.Drive();  // Own method

2. Multilevel Inheritance

public class LivingThing
{
    public void Grow() => Console.WriteLine("Growing...");
}

public class Animal : LivingThing
{
    public void Move() => Console.WriteLine("Moving...");
}

public class Bird : Animal
{
    public void Fly() => Console.WriteLine("Flying...");
}

// Usage
Bird b = new Bird();
b.Grow();  // From LivingThing
b.Move();  // From Animal
b.Fly();   // Own method

3. Hierarchical Inheritance

public class Shape
{
    public void Draw() => Console.WriteLine("Drawing shape...");
}

public class Circle : Shape
{
    public void Radius() => Console.WriteLine("Circle radius...");
}

public class Square : Shape
{
    public void Side() => Console.WriteLine("Square side...");
}

üõ°Ô∏è Sealed Classes

The sealed keyword prevents a class from being inherited. It's useful when you want to restrict further extension of a class for security, design, or performance reasons.

public sealed class PaymentProcessor
        {
            public void Process() => Console.WriteLine("Processing payment...");
        }

        // ‚ùå This will cause a compile-time error
        public class CustomProcessor : PaymentProcessor
        {
            public void Customize() => Console.WriteLine("Custom logic...");
        }

Attempting to inherit from a sealed class results in a compile-time error.

‚úÖ Sealing Methods

You can also seal methods in a derived class to prevent further overriding:

public class BaseLogger
        {
            public virtual void Log() => Console.WriteLine("Base log");
        }

        public class FileLogger : BaseLogger
        {
            public sealed override void Log() => Console.WriteLine("File log");
        }

        // ‚ùå Cannot override Log() in further derived classes
        public class SecureLogger : FileLogger
        {
            // Compile-time error: cannot override sealed method
            // public override void Log() => Console.WriteLine("Secure log");
        }
üí° Educator Tip: Use sealed classes to demonstrate design constraints and encourage thoughtful architecture. Ask students when and why they'd seal a class or method.

üìò Summary

  • Inheritance allows classes to reuse and extend functionality.
  • Use : to inherit from a base class.
  • Types include single, multilevel, and hierarchical inheritance.
  • Derived classes can access public and protected members of the base class.
üí° Educator Tip: Ask students to build a class hierarchy for a zoo or vehicle system to practice inheritance.
