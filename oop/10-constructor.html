üîç Overview

Constructors are special methods that execute when an object is created. They ensure fields start with valid values and centralize setup logic.

This session covers:

  • Constructor basics and types
  • Benefits of using constructors
  • Field-only examples (no properties)

üìê Definition

A constructor has the same name as its class, no return type, and runs on new. If you don‚Äôt write one, the compiler provides a default.

Constructor variants:

  • Default (parameterless)
  • Parameterized
  • Overloaded
  • Static (for initializing static fields)

üèÜ Advantages of Using Constructors

  • Guarantees valid initial state for all instances
  • Centralizes initialization logic in one place
  • Supports overloads for flexible object creation
  • Enforces required data through parameters
  • Keeps field setup encapsulated within the class

üíª Examples (Field-Only)

1. Default Constructor

public class Book
{
    // Fields
    public string title;
    public int pages;

    // Default constructor
    public Book()
    {
        title = "Untitled";
        pages = 0;
    }
}

// Usage
var b1 = new Book();
Console.WriteLine(b1.title + ", " + b1.pages + " pages");

2. Parameterized Constructor

public class Book
{
    public string title;
    public int pages;

    // Parameterized constructor
    public Book(string title, int pages)
    {
        this.title = title;
        this.pages = pages;
    }
}

// Usage
var b2 = new Book("1984", 328);
Console.WriteLine(b2.title + ", " + b2.pages + " pages");

3. Constructor Overloading

public class Book
{
    public string title;
    public int pages;

    public Book() : this("Untitled", 0) { }

    public Book(string title) : this(title, 0) { }

    public Book(string title, int pages)
    {
        this.title = title;
        this.pages = pages;
    }
}

// Usage
var a = new Book();
var b = new Book("Brave New World");
var c = new Book("Dune", 412);

Console.WriteLine(a.title + ", " + a.pages);
Console.WriteLine(b.title + ", " + b.pages);
Console.WriteLine(c.title + ", " + c.pages);
üí° Educator Tip: Have students add manual validation in the constructor (e.g., ensure pages > 0) and throw exceptions on invalid data.

üìò Summary

  • Constructors initialize fields without using properties.
  • Types include default, parameterized, overloaded, and static.
  • Key benefits: valid state, centralized logic, and flexible instantiation.
  • Field-based examples show direct assignment and overload chaining.
