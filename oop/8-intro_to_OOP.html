üîç Overview

Object-Oriented Programming (OOP) is a paradigm that models software as a collection of objects. Each object encapsulates data (attributes) and behavior (methods).

Key principles include:

  • Encapsulation: Bundling data with methods that operate on that data.
  • Abstraction: Hiding complex implementation details behind a simple interface.
  • Inheritance: Deriving new classes from existing ones to promote code reuse.
  • Polymorphism: Allowing objects to be treated as instances of their parent class.

üèÜ Advantages of OOP

  • Modularity: Objects serve as self-contained modules.
  • Reusability: Inheritance and polymorphism let you reuse existing code.
  • Maintainability: Easier to update and extend modular objects.
  • Scalability: Simplifies large-scale system design via componentization.
  • Robustness: Encapsulation protects object state.
üí° Educator Tip: Use real-world analogies (e.g., Car objects, Person objects) to ground these abstract benefits.

üö´ DRY Concept

DRY stands for Don't Repeat Yourself. In OOP, methods and classes help eliminate duplicated logic.

For example, without DRY you might see:

// Repeated code without DRY
Console.WriteLine("Welcome, Alice!");
Console.WriteLine("Welcome, Bob!");
Console.WriteLine("Welcome, Carol!");

Applying DRY with a method:

void Greet(string name) {
  Console.WriteLine("Welcome, " + name + "!");
}

Greet("Alice");
Greet("Bob");
Greet("Carol");

{
  // Attribute
  public string Name { get; set; }

  // Constructor
  public Person(string name)
  {
    Name = name;
  }

  // Method
  public void Introduce()
  {
    Console.WriteLine($"Hi, my name is {Name}.");
  }
}

// Usage in Main()
Person alice = new Person("Alice");
alice.Introduce();  // Output: Hi, my name is Alice.

üìò Summary

  • OOP organizes code into objects that bundle data and behavior.
  • Advantages include modularity, reusability, maintainability, scalability, and robustness.
  • DRY principle reduces duplication via methods and classes.
  • Defining a simple class with properties and methods is the first step in OOP.
