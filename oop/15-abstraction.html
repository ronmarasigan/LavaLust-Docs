üß† Concept

Abstraction in C# lets you define a simplified model of a complex system. You expose only the essential features and hide implementation details, leading to cleaner, more maintainable code.

  • Focus on what an object does, not how it does it.
  • Encourages a clear separation between interface and implementation.
  • Implemented via abstract classes and methods.

üéØ Why Use Abstraction?

  • Reduces complexity by hiding details.
  • Improves reusability‚Äîcommon code in base class.
  • Enforces a contract‚Äîderived classes must implement required methods.
  • Enhances extensibility and testability.

üìê Abstract Classes

An abstract class acts as a blueprint. It can contain fields, constructors, concrete methods, and abstract members. You cannot instantiate it directly.

public abstract class Animal
{
    public string Name { get; }
    public Animal(string name)
    {
        Name = name;
    }

    // Concrete method
    public void Sleep()
    {
        Console.WriteLine($"{Name} is sleeping.");
    }

    // Abstract member
    public abstract void Speak();
}
üí° Tip: Use abstract classes when multiple related types share code but also require specific behavior.

‚úçÔ∏è Abstract Methods

An abstract method has no body. Every subclass must override it, ensuring a consistent interface.

public class Dog : Animal
{
    public Dog(string name) : base(name) { }

    // Required override
    public override void Speak()
    {
        Console.WriteLine($"{Name} says: Woof!");
    }
}

public class Cat : Animal
{
    public Cat(string name) : base(name) { }

    public override void Speak()
    {
        Console.WriteLine($"{Name} says: Meow!");
    }
}
üí° Tip: The compiler enforces overrides‚Äîmissing an abstract method causes a compile-time error.

üíª Example

// Cannot do: new Animal("Rex"); // Error: cannot create instance of abstract class

Animal[] pets = {
    new Dog("Rex"),
    new Cat("Mittens")
};

foreach (var pet in pets)
{
    pet.Speak(); // Rex says: Woof!  then  Mittens says: Meow!
    pet.Sleep(); // Rex is sleeping.    then  Mittens is sleeping.
}

This demonstrates:

  • Abstract base Animal enforces Speak().
  • Derived classes provide concrete implementations.
  • Shared behavior (Sleep()) lives in base class.
üí° Tip: Ask students to add a new subclass (e.g., Bird) to practice compile-time enforcement.

üìò Summary

  • Abstract classes cannot be instantiated; they define blueprints.
  • They can include both concrete and abstract members.
  • Abstract methods declare required behavior; subclasses must override.
  • Use abstraction to design clear, extensible hierarchies.
