Enums in C#

Named constants for improved readability and safety

üß† Concept

Enums (enumerations) define a set of named integral constants. They improve code clarity by replacing magic numbers with descriptive names.

  • Group related constants under one type
  • Enhance type safety compared to raw integers
  • Make switch statements and comparisons more readable

üìê Declaration

Use the enum keyword followed by a comma-separated list of identifiers.

public enum DayOfWeek
{
    Sunday,
    Monday,
    Tuesday,
    Wednesday,
    Thursday,
    Friday,
    Saturday
}
üí° Tip: By default, the first member has value 0, and each subsequent member increments by 1.

üî¢ Underlying Values

You can specify explicit values or change the underlying integer type.

public enum ErrorCode : byte
{
    None = 0,
    NotFound = 1,
    PermissionDenied = 128,
    Unknown = 255
}
  • Default underlying type is int.
  • Use : to choose another integral type (byte, short, etc.).

üíª Example

// Declare an enum
public enum TrafficLight
{
    Red,
    Yellow,
    Green
}

// Use it in code
TrafficLight signal = TrafficLight.Red;

switch (signal)
{
    case TrafficLight.Red:
        Console.WriteLine("Stop"); break;
    case TrafficLight.Yellow:
        Console.WriteLine("Get Ready"); break;
    case TrafficLight.Green:
        Console.WriteLine("Go"); break;
}

// Output: Stop
üí° Tip: Enums can be combined with [Flags] attribute for bitwise operations.

üìò Summary

  • Enums provide named, integral constants for clarity and safety.
  • Default values start at 0 and increment by 1 unless overridden.
  • You can specify an underlying type and explicit values.
  • Ideal for switch statements and representing fixed sets of options.
üí° Educator Tip: Ask students to define enums for month names or permission flags and use them in conditional logic.
