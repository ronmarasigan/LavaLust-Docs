üß† Concept

Interfaces declare a contract of members (methods, properties, events) without implementation. They specify what a type must do, not how to do it.

  • Enable polymorphism across unrelated types
  • Support multiple inheritance of behavior
  • Promote loose coupling and testability

üéØ Why Use Interfaces?

  • Enforce consistent APIs across different classes
  • Allow swapping implementations without changing clients
  • Facilitate mocking and unit testing
  • Encourage separation of concerns

üìê Defining an Interface

Use the interface keyword. Members have no bodies and are implicitly public.

public interface IPlayable
{
    void Play();
    void Stop();
    int Position { get; set; }
}
üí° Tip: No fields or constructors‚Äîonly methods, properties, events, and indexers are allowed.

üîß Implementing Interfaces

Classes or structs implement interfaces with the : symbol and must provide all member definitions.

public class Song : IPlayable
{
    public int Position { get; set; }

    public void Play() =>
        Console.WriteLine("Playing song at position " + Position);

    public void Stop() =>
        Console.WriteLine("Song stopped.");
}

public class Video : IPlayable
{
    public int Position { get; set; }

    public void Play() =>
        Console.WriteLine("Playing video at position " + Position);

    public void Stop() =>
        Console.WriteLine("Video stopped.");
}

üíª Example

IPlayable[] playlist = {
    new Song { Position = 0 },
    new Video { Position = 10 }
};

foreach (var item in playlist)
{
    item.Play();
    item.Stop();
}

This shows how different types share the same interface and can be used interchangeably.

üí° Tip: Ask students to add a new IPlayable implementation (e.g., Podcast) to practice.

üìò Summary

  • Interfaces declare a set of members without implementation.
  • Types implement interfaces to fulfill contracts.
  • Support polymorphism and multiple inheritance of behavior.
  • Use interfaces for flexibility, testability, and clear API design.
üí° Educator Tip: Model a scenario (e.g., payment processors) with an interface to demonstrate interchangeable implementations.
