üß¨ Exception Hierarchy

All exceptions in C# derive from the base class System.Exception. This hierarchy allows structured handling and polymorphism.

  • System.Object ‚Äì Root of all types in .NET
  • System.Exception ‚Äì Base class for all exceptions
  • System.SystemException ‚Äì Base for runtime exceptions
  • ApplicationException ‚Äì Optional base for user-defined exceptions

Note: You can catch exceptions at any level depending on how specific your handling needs to be.

üì¶ Built-in Exceptions

DivideByZeroException

Occurs when dividing a number by zero.

int x = 10;
int y = 0;
int result = x / y; // Throws DivideByZeroException

NullReferenceException

Occurs when accessing a member of a null object.

string name = null;
Console.WriteLine(name.Length); // Throws NullReferenceException

IndexOutOfRangeException

Occurs when accessing an array index that doesn't exist.

int[] numbers = {1, 2, 3};
Console.WriteLine(numbers[5]); // Throws IndexOutOfRangeException

FileNotFoundException

Occurs when trying to access a file that doesn't exist.

using System.IO;
File.ReadAllText("missing.txt"); // Throws FileNotFoundException

FormatException

Occurs when converting a string to a number with invalid format.

string input = "abc";
int number = int.Parse(input); // Throws FormatException

üõ†Ô∏è User-Defined Exceptions

Create custom exceptions by inheriting from Exception or ApplicationException.

public class InvalidGradeException : Exception {
    public InvalidGradeException(string message) : base(message) { }
}

Usage:

int grade = -10;

if (grade < 0 || grade > 100) {
    throw new InvalidGradeException("Grade must be between 0 and 100.");
}

üö® Throwing Exceptions

Use throw to raise an exception manually when a condition violates expected logic.

throw new ArgumentException("Invalid argument passed.");

Tip: Always provide meaningful messages and use specific exception types.

‚úÖ Assertions

Definition: Assertions are used to validate assumptions during development. They help catch logical errors early.

Namespace: System.Diagnostics

using System.Diagnostics;
int age = 25; Debug.Assert(age >= 0, "Age cannot be negative.");

Use Case: Assertions are ideal for internal checks during development and debugging. They do not throw exceptions in release builds but help catch logic errors early in debug mode.

Note: Assertions should not replace exception handling‚Äîthey complement it by verifying conditions that should never fail.

  • User-defined exceptions allow domain-specific error signaling.
  • throw is used to raise exceptions manually.
  • Debug.Assert helps validate assumptions during development.
