Definition, purpose, and practical examples

üîç Overview

Properties in C# provide a flexible mechanism to read, write, or compute the value of a private field. They act as public access points while maintaining encapsulation.

This lecture covers:

  • What properties are and why they matter
  • Types of properties
  • Field-backed and computed examples

üìê Definition

A property is a member that provides controlled access to a class field. It typically includes get and set accessors.

private string name;

public string Name
{
    get { return name; }
    set { name = value; }
}

Properties can be auto-implemented or manually backed by fields.

üîß Types of Properties

  • Auto-Implemented: Simplified syntax with implicit backing field
  • Read-Only: Only get accessor
  • Write-Only: Only set accessor
  • Computed: No backing field; returns calculated value
üí° Educator Tip: Use computed properties to demonstrate encapsulated logic (e.g., BMI, full name).

üíª Examples

1. Auto-Implemented Property

public class Book
{
    public string Title { get; set; }
    public int Pages { get; set; }
}

// Usage
Book b = new Book();
b.Title = "1984";
b.Pages = 328;
Console.WriteLine(b.Title + ", " + b.Pages + " pages");

2. Field-Backed Property

public class Person
{
    private string name;

    public string Name
    {
        get { return name; }
        set { name = value; }
    }
}

// Usage
Person p = new Person();
p.Name = "Alice";
Console.WriteLine("Name: " + p.Name);

3. Read-Only Property

public class Circle
{
    private double radius;

    public Circle(double r)
    {
        radius = r;
    }

    public double Radius
    {
        get { return radius; }
    }
}

4. Computed Property

public class Rectangle
{
    public int Width { get; set; }
    public int Height { get; set; }

    public int Area
    {
        get { return Width * Height; }
    }
}

// Usage
Rectangle r = new Rectangle { Width = 5, Height = 4 };
Console.WriteLine("Area: " + r.Area);

üìò Summary

  • Properties provide controlled access to fields using get and set.
  • They support encapsulation and validation logic.
  • Types include auto-implemented, read-only, write-only, and computed.
  • Examples show how to use properties for both simple and derived values.
