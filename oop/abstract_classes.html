üìò What Is an Abstract Class?

An abstract class is a blueprint for other classes. It cannot be instantiated directly and may contain abstract methods that must be implemented by child classes.

üõ† Syntax

abstract class Animal {
    public $name;

    public function __construct($name) {
        $this->name = $name;
    }

    abstract public function makeSound(): string;
}

üßí Child Class Implementation

class Dog extends Animal {
    public function makeSound(): string {
        return "Woof! I'm a {$this->name}";
    }
}

$dog = new Dog("Buddy");
echo $dog->makeSound(); // Output: Woof! I'm a Buddy

üìã Rules

  • Abstract classes cannot be instantiated.
  • Any class with an abstract method must be abstract.
  • Child classes must implement all abstract methods.
  • Access modifiers must be the same or less restrictive (e.g., protected ‚Üí public).
  • Method signatures must match (same name, required parameters).

üß≠ Abstract Methods with Arguments

abstract class Formatter {
    abstract protected function format($text);
}

class HtmlFormatter extends Formatter {
    public function format($text) {
        return "<p>" . htmlspecialchars($text) . "</p>";
    }
}

üèõ Abstract Classes with Constructors

abstract class Base {
    public function __construct() {
        echo "Base constructor\n";
    }

    abstract public function describe();
}

class Derived extends Base {
    public function __construct() {
        parent::__construct();
        echo "Derived constructor\n";
    }

    public function describe() {
        echo "This is a derived class\n";
    }
}
