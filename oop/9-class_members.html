Fields

A field is a variable declared directly in a class, representing private or public data.

public class Car
{
    // Field
    private string model;
}

Properties

Properties expose fields with controlled access (getters/setters).

public class Car
{
    // Auto-implemented property
    public string Model { get; set; }
}

Constructors

Constructors initialize new objects. They share the class name and have no return type.

public Car(string model)
{
    Model = model;
}

Methods

Methods define actions or behaviors that objects can perform.

public void Drive()
{
    Console.WriteLine($"Driving a {Model}");
}
üí° Educator Tip: Ask students to identify fields vs. properties in existing class code to reinforce access control.

üë• Multiple Objects

One class can spawn any number of objects (instances). Each object maintains its own state.

Example: Creating three Car objects:

Car car1 = new Car("Sedan");
Car car2 = new Car("SUV");
Car car3 = new Car("Convertible");

car1.Drive();  // Driving a Sedan
car2.Drive();  // Driving a SUV
car3.Drive();  // Driving a Convertible
üí° Educator Tip: Challenge students to store objects in a List and iterate to call methods.

üíª Examples

Person Class with Multiple Members

public class Person
{
    // Fields
    private int age;

    // Property
    public string Name { get; private set; }

    // Constructor
    public Person(string name, int age)
    {
        Name = name;
        this.age = age;
    }

    // Method
    public void Introduce()
    {
        Console.WriteLine($"Hi, I'm {Name}, {age} years old.");
    }
}

// In Main()
var people = new List
{
    new Person("Alice", 30),
    new Person("Bob", 25),
    new Person("Cara", 28)
};

foreach (var p in people)
{
    p.Introduce();
}
üí° Educator Tip: Extend this by adding a Birthday() method that increments age and prints a celebration message.

üìò Summary

  • Fields, properties, constructors, and methods make up class members.
  • Fields hold raw data; properties control access to that data.
  • Constructors set initial state; methods define behavior.
  • You can create many objects from one class; each has its own state.
