üîç Overview

Access modifiers control the visibility and accessibility of classes, fields, methods, and other members in C#. They help enforce encapsulation and protect internal logic.

This lecture covers:

  • What access modifiers are
  • Common types and their scopes
  • Field-based examples for clarity

üìê Definition

An access modifier is a keyword that defines how and where a class member can be accessed. It determines the level of exposure to other parts of the program.

Modifiers are placed before the member declaration:

public int age;
private string name;

üîì Types of Access Modifiers

  • public ‚Äì Accessible from anywhere
  • private ‚Äì Accessible only within the same class
  • protected ‚Äì Accessible within the class and derived classes
  • internal ‚Äì Accessible within the same assembly
  • protected internal ‚Äì Accessible within the same assembly or derived classes
  • private protected ‚Äì Accessible within the same class or derived classes in the same assembly

üíª Examples

1. Using public and private

public class Person
{
    public string name;       // Accessible from anywhere
    private int age;          // Accessible only within Person

    public Person(string name, int age)
    {
        this.name = name;
        this.age = age;
    }

    public void ShowInfo()
    {
        Console.WriteLine("Name: " + name);
        Console.WriteLine("Age: " + age);  // Valid: age is private but accessed inside class
    }
}

// Usage
Person p = new Person("Alice", 30);
Console.WriteLine(p.name);    // Valid
// Console.WriteLine(p.age);  // Error: age is private

2. Using protected

public class Animal
{
    protected string species;

    public Animal(string species)
    {
        this.species = species;
    }
}

public class Dog : Animal
{
    public Dog(string species) : base(species) {}

    public void Bark()
    {
        Console.WriteLine("Species: " + species);  // Valid: protected accessible in derived class
    }
}

3. Using internal

internal class Engine
{
    public void Start()
    {
        Console.WriteLine("Engine started.");
    }
}

// Accessible within the same assembly
Engine e = new Engine();
e.Start();

üìò Summary

  • Access modifiers define visibility and control access to class members.
  • They support encapsulation and protect internal logic.
  • Common modifiers: public, private, protected, internal, protected internal, private protected.
  • Examples show how modifiers affect access across classes and inheritance.
