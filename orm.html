Class Overview

The Model class provides an Object-Relational Mapping (ORM) implementation for LavaLust PHP Framework. It offers:

  • Simple and intuitive database interaction
  • Built-in support for soft deletes
  • CRUD operations with minimal code
  • Advanced query building capabilities
  • Relationship management (hasOne, hasMany, belongsTo, manyToMany)
  • Pagination and result limiting
  • Transaction support

Basic Usage

Extend the Model class to create your own models:

class User extends Model {
    protected $table = 'users';
    protected $primary_key = 'id';
    protected $fillable = ['name', 'email', 'password'];
}

Model Properties

protected $table
string

The database table associated with the model.

protected $primary_key
string

The primary key for the table (default: 'id').

protected $fillable
array

Attributes that are mass assignable (for create/update).

protected $soft_delete_column
string

Column name to use for soft deletes (default: 'deleted_at').

protected $has_soft_delete
boolean

Whether soft deletes are enabled for this model.

CRUD Operations

insert($data)

Inserts a new record into the database.

Parameters

Parameter Type Description
$data array Associative array of column => value

Return Value

integer - ID of the inserted record

Example

$user_id = $this->user->insert([
    'name' => 'John Doe',
    'email' => 'john@example.com',
    'password' => password_hash('secret', PASSWORD_DEFAULT)
]);

update($id, $data)

Updates an existing record in the database.

Parameters

Parameter Type Description
$id integer ID of the record to update
$data array Associative array of column => value

Return Value

integer - Number of affected rows

Example

$updated = $this->user->update(1, [
    'name' => 'Jane Doe',
    'email' => 'jane@example.com'
]);

delete($id)

Deletes a record from the database (permanently).

Parameters

Parameter Type Description
$id integer ID of the record to delete

Return Value

integer - Number of affected rows

Example

$deleted = $this->user->delete(1);

Parameter Type Description Default

Return Value

array|null - Last record or null

Parameters

Parameters

Parameter Type Description

Return Value

integer - Number of affected rows

Example

Enabling Soft Deletes

To use soft deletes, set these properties in your model:

protected $has_soft_delete = true;
protected $soft_delete_column = 'deleted_at';

And add a deleted_at (or your custom column) to your table:

ALTER TABLE users ADD deleted_at DATETIME NULL;

Bulk Operations

bulk_insert($data)

Inserts multiple records in a single query.

Parameters

Parameter Type Description
$data array Array of associative arrays

Return Value

integer - Number of inserted rows

Example

$users = [
    ['name' => 'John', 'email' => 'john@example.com'],
    ['name' => 'Jane', 'email' => 'jane@example.com'],
    ['name' => 'Bob', 'email' => 'bob@example.com']
];

$inserted = $this->user->bulk_insert($users);

bulk_update($data, $key = 'id')

Updates multiple records in a single query.

Parameters

Parameter Type Description Default
$data array Array of associative arrays Required
$key string Column to use as the update key 'id'

Return Value

integer - Number of updated rows

Example

$updates = [
    ['id' => 1, 'status' => 'active'],
    ['id' => 2, 'status' => 'inactive'],
    ['id' => 3, 'status' => 'active']
];

$updated = $this->user->bulk_update($updates);

Model Relationships

has_one($related_model, $foreign_key, $current_id)

Defines a one-to-one relationship.

Parameters

Parameter Type Description
$related_model string Name of the related model
$foreign_key string Foreign key in the related table
$current_id mixed ID of the current model

Example

// In User model
public function profile($user_id) {
    return $this->has_one('Profile', 'user_id', $user_id);
}

// Usage
$user_profile = $this->user->profile(1);

has_many($related_model, $foreign_key, $current_id)

Defines a one-to-many relationship.

Example

// In User model
public function posts($user_id) {
    return $this->has_many('Post', 'user_id', $user_id);
}

// Usage
$user_posts = $this->user->posts(1);

belongs_to($related_model, $foreign_key, $foreign_key_value)

Defines an inverse relationship.

Example

// In Post model
public function author($post_id) {
    $post = $this->find($post_id);
    return $this->belongs_to('User', 'user_id', $post['user_id']);
}

// Usage
$post_author = $this->post->author(123);

many_to_many($related_model, $pivot_table, $current_key, $related_key, $current_id)

Defines a many-to-many relationship.

Example

// In User model
public function roles($user_id) {
    return $this->many_to_many('Role', 'user_roles', 'user_id', 'role_id', $user_id);
}

// Usage
$user_roles = $this->user->roles(1);

Pagination

paginate($per_page = 10, $page = 1, $conditions = [], $with_deleted = false)

Paginates query results.

Parameters

Parameter Type Description Default
$per_page integer Items per page 10
$page integer Current page number 1
$conditions array WHERE conditions []
$with_deleted boolean Include soft-deleted records false

Return Value

array - Pagination data structure

Example

$pagination = $this->user->paginate(10, 1, ['status' => 'active']);

// Returns:
[
    'data' => [...],         // Current page results
    'total' => 100,          // Total records
    'per_page' => 10,        // Items per page
    'current_page' => 1,     // Current page
    'last_page' => 10        // Total pages
]

Transactions

transaction()

Starts a database transaction.

Example

$this->user->transaction();

commit()

Commits the current transaction.

Example

$this->user->commit();

rollback()

Rolls back the current transaction.

Example

$this->user->rollback();

Transaction Example

try {
    $this->user->transaction();
    
    $user_id = $this->user->insert([...]);
    $this->profile->insert(['user_id' => $user_id, ...]);
    
    $this->user->commit();
} catch(Exception $e) {
    $this->user->rollback();
    // Handle error
}

Best Practices

Model Structure

class User extends Model {
    // Database table
    protected $table = 'users';
    
    // Primary key
    protected $primary_key = 'id';
    
    // Fillable fields for mass assignment
    protected $fillable = ['name', 'email', 'password'];
    
    // Soft delete configuration
    protected $has_soft_delete = true;
    protected $soft_delete_column = 'deleted_at';
}

Security Considerations

  • Always use the $fillable property to prevent mass assignment vulnerabilities
  • Validate all input data before inserting or updating
  • Use parameter binding in raw queries
  • Limit the use of $with_deleted = true to admin areas only

Performance Tips

  • Use select() to specify only needed columns
  • Add indexes to frequently queried columns
  • For large datasets, use pagination instead of all()
  • Use bulk operations for multiple inserts/updates
