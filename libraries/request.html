Request Class

The Request class handles HTTP requests, input data (GET, POST, COOKIE, SERVER), CSRF protection, and request utilities. It provides a clean interface for accessing request data and checking request properties.

Loading the Request Class

The Request class is automatically loaded by the LavaLust framework and is accessible via:

<?php
$this->request;

No manual loading is required as it’s available globally.

CSRF Protection

If CSRF protection is enabled in your configuration (csrf_protection = TRUE), the Request class will automatically validate CSRF tokens on POST requests. Invalid requests will be rejected.

Class Methods

Input Methods

post()

Parameter

Type

Default

Description

$index

string|null

NULL

Specific key to retrieve, or NULL for all POST data

Returns: mixed (string, array, or NULL)

Description: Retrieves POST variables. Returns all POST data as array when no index is specified.

Example:

<?php
// Get all POST data
$all_post = $this->request->post();

// Get specific POST field
$username = $this->request->post('username');

// Check if POST field exists
if ($this->request->post('email') !== NULL) {
    // Process email
}

Request Information Methods

method()

Parameter

Type

Default

Description

$upper

boolean

FALSE

If TRUE, returns uppercase method name

Returns: string

Description: Returns the HTTP request method (GET, POST, PUT, PATCH, DELETE, etc.).

Example:

<?php
$method = $this->request->method(); // Returns 'get', 'post', etc.
$method_upper = $this->request->method(TRUE); // Returns 'GET', 'POST', etc.

if ($this->request->method(TRUE) === 'POST') {
    // Handle POST request
}

Parameter

Type

Default

Description

string|array

—

Single method or array of methods to check

if ($this->request->is_method('post')) {

if ($this->request->is_method(array('put', 'patch'))) {

}

wants_json()

Parameters: None

Returns: boolean

Description: Checks if the client expects a JSON response (via Accept header).

Example:

<?php
if ($this->request->wants_json()) {
    $this->response->json($data);
} else {
    $this->call->view('html_view', $data);
}

is_secure()

Parameters: None

Returns: boolean

Description: Checks if the request is using HTTPS.

Example:

<?php
if ( ! $this->request->is_secure()) {
    redirect(str_replace('http://', 'https://', $this->request->url()));
}

ip_address()

Parameter

Type

Default

Description

$trust_proxy

boolean

FALSE

Set to TRUE if behind a trusted proxy

Returns: string

Description: Gets the client’s IP address. Only trusts proxy headers when explicitly enabled.

Security Note: Never trust proxy headers by default. Only enable $trust_proxy when you control the proxy server.

Example:

<?php
// Basic usage (does not trust proxy headers)
$user_ip = $this->request->ip_address();

// If behind trusted proxy (e.g., Cloudflare, Nginx)
$user_ip = $this->request->ip_address(TRUE);

Complete Examples

Form Processing with Validation:

<?php
class UserController extends Controller {

    public function register() {
        if ($this->request->is_post()) {
            // Get validated input
            $data = $this->request->only(['username', 'email', 'password']);

            // CSRF is automatically handled if enabled

            if (empty($data['username'])) {
                $errors[] = 'Username is required';
            }

            if ( ! filter_var($data['email'], FILTER_VALIDATE_EMAIL)) {
                $errors[] = 'Invalid email address';
            }

            if (empty($errors)) {
                // Save user
                $this->user_model->create($data);
                $this->response->redirect('login');
            }
        }

        $this->call->view('register_form');
    }
}

RESTful API Endpoint:

<?php
class ApiController extends Controller {

    public function users($id = NULL) {
        $method = $this->request->method(TRUE);

        switch ($method) {
            case 'GET':
                if ($id) {
                    $data = $this->user_model->find($id);
                } else {
                    $data = $this->user_model->all();
                }
                break;

            case 'POST':
                $data = $this->request->post();
                $result = $this->user_model->create($data);
                break;

            case 'PUT':
                $data = $this->request->put();
                $result = $this->user_model->update($id, $data);
                break;

            case 'DELETE':
                $result = $this->user_model->delete($id);
                break;
        }

        if ($this->request->wants_json()) {
            $this->response->json($data);
        }
    }
}

Secure Cookie Management:

<?php
class AuthController extends Controller {

    public function login() {
        if ($this->request->is_post()) {
            $credentials = $this->request->only(['username', 'password']);

            if ($user = $this->auth_model->authenticate($credentials)) {
                // Set secure session cookie
                $token = bin2hex(random_bytes(32));

                $this->request->set_cookie('auth_token', $token, 86400, array(
                    'httponly' => TRUE,
                    'secure' => $this->request->is_secure(),
                    'samesite' => 'Strict'
                ));

                $this->response->redirect('dashboard');
            }
        }
    }

    public function logout() {
        $this->request->delete_cookie('auth_token');
        $this->response->redirect('login');
    }
}

Best Practices

  • Always validate and sanitize input before using it in your application

  • Enable CSRF protection for all forms that modify data

  • Use ``only()`` and ``except()`` to prevent mass assignment vulnerabilities

  • Don’t trust proxy headers unless you control the proxy server

  • Use secure cookie flags (httponly, secure, samesite) for sensitive data

  • Check request methods to ensure proper handling (e.g., only accept POST for form submission)

  • Use ``wants_json()`` to serve appropriate response formats

  • Never output raw user input without escaping to prevent XSS attacks

Security Considerations

Warning

The Request class provides raw input data. You must sanitize and validate all data before using it in database queries, file operations, or output.

Built-in Security Features:

  • CSRF protection when enabled in configuration

  • Cookie name validation to prevent injection

  • IP validation with proper filtering

  • Proxy header trust disabled by default

  • HTTPS detection for secure cookies

Security Checklist:

<?php
// DO: Validate and sanitize input
$email = filter_var($this->request->post('email'), FILTER_VALIDATE_EMAIL);
$name = htmlspecialchars($this->request->post('name'), ENT_QUOTES);

// DON'T: Trust raw input directly
$query = "SELECT * FROM users WHERE id = " . $this->request->get('id');

// DO: Use prepared statements or escaping
$id = (int) $this->request->get('id');
$query = "SELECT * FROM users WHERE id = " . $id;

Troubleshooting

Common Issues and Solutions

Issue

Possible Solution

POST data not available

Ensure your form has method="post" and CSRF token is included if enabled

PUT/DELETE data not received

Check that your server supports these methods and the client sends proper Content-Type

Cookie not being set

Verify cookie name has no invalid characters. Check that headers haven’t been sent yet

AJAX detection failing

Ensure your JavaScript sets X-Requested-With: XMLHttpRequest header

Wrong IP address detected

Only enable $trust_proxy if behind a trusted proxy. Check your proxy configuration

CSRF validation fails

Make sure your forms include the CSRF token field generated by the Security class
