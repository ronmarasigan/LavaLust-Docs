Response Class

The Response class handles HTTP responses, status codes, headers, content types, and provides convenient methods for common response formats like JSON, HTML, XML, file downloads, and redirects.

Loading the Response Class

The Response class is automatically loaded by the LavaLust framework and is accessible via:

<?php
$this->response;

No manual loading is required as it’s available globally.

Class Methods

Status Code Methods

set_status_code()

Parameter

Type

Default

Description

$status_code

int

—

HTTP status code (200, 404, 500, etc.)

Returns: object (method chaining)

Description: Sets the HTTP status code for the response.

Example:

<?php
$this->response->set_status_code(404);
$this->response->set_status_code(201)->send_json($data);

get_status_code()

Parameters: None

Returns: int|null

Description: Returns the current HTTP status code.

Example:

<?php
$code = $this->response->get_status_code();
if ($code === 200) {
    // Success
}

Header Methods

add_header()

Parameter

Type

Default

Description

$name

string|array

—

Header name or associative array of headers

$value

string

‘’

Header value (required if $name is string)

Returns: object (method chaining)

Description: Adds one or more HTTP headers to the response.

Example:

<?php
// Single header
$this->response->add_header('Content-Type', 'application/json');

// Multiple headers
$this->response->add_header(array(
    'X-Custom-Header' => 'value',
    'Cache-Control' => 'no-cache'
));

remove_header()

Parameter

Type

Default

Description

$name

string

—

Header name to remove

Returns: object (method chaining)

Description: Removes a previously set header.

Example:

<?php
$this->response->remove_header('X-Debug-Info');

get_headers()

Parameters: None

Returns: array

Description: Returns all currently set headers.

Example:

<?php
$headers = $this->response->get_headers();
foreach ($headers as $name => $value) {
    echo "$name: $value";
}

send()

Parameters: None

Returns: void

Description: Sends the response including status code, headers, and content.

Example:

<?php
$this->response->set_content('Hello World')
               ->add_header('X-Powered-By', 'LavaLust')
               ->send();

File Methods

string

—

Path to the file to download

$filename

string

Custom filename for download

$headers

array

Additional headers to send

Returns: void

Description: Forces a file download.

Example:

$this->response->download('/path/to/file.pdf', 'invoice.pdf');

stream()

Parameter

Type

Default

Description

$callback

callable

—

Function that outputs content

$headers

array

array()

Additional headers to send

Returns: void

Description: Streams content to the browser in chunks.

Example:

<?php
// Stream large file
$this->response->stream(function() {
    $handle = fopen('/path/to/large-file.log', 'rb');
    while (!feof($handle)) {
        echo fread($handle, 8192);
        flush();
    }
    fclose($handle);
}, array('Content-Type' => 'text/plain'));

Utility Methods

send_headers()

Parameters: None

Returns: void

Description: Sends all queued headers and cookies without sending content.

Example:

<?php
$this->response->add_header('X-Custom', 'value')
               ->send_headers();
// Continue with custom output

clear()

Parameters: None

Returns: object (method chaining)

Description: Clears all response data (status code, headers, content, cookies).

Example:

<?php
$this->response->set_content('Old data')
               ->clear()
               ->set_content('New data')
               ->send();

__toString()

Parameters: None

Returns: string

Description: Magic method that returns response content as string when the object is treated as a string.

Example:

<?php
echo $this->response; // Outputs response content

Complete Examples

Basic JSON API Endpoint:

<?php
class ApiController extends Controller {

    public function users() {
        $users = $this->user_model->get_all();

        if ($users) {
            $this->response->send_json_success($users);
        } else {
            $this->response->send_json_error('No users found', 404);
        }
    }

    public function create_user() {
        $data = $this->request->post();

        if ($this->validate_user($data)) {
            $id = $this->user_model->create($data);
            $this->response->send_json_success(
                array('id' => $id),
                'User created successfully',
                201
            );
        } else {
            $this->response->send_json_error('Invalid user data', 422);
        }
    }
}

File Upload with Response:

<?php
class FileController extends Controller {

    public function upload() {
        if ( ! $this->request->is_post()) {
            $this->response->set_status_code(405)
                           ->send_json_error('Method not allowed', 405);
            return;
        }

        $this->call->library('upload', $_FILES['file']);
        $this->upload->set_dir('uploads')->allowed_extensions(['jpg', 'png']);

        if ($this->upload->do_upload()) {
            $this->response->send_json_success(array(
                'filename' => $this->upload->get_filename(),
                'size' => $this->upload->get_size()
            ), 'File uploaded successfully');
        } else {
            $this->response->send_json_error($this->upload->get_errors(), 400);
        }
    }

    public function download($filename) {
        $filepath = 'uploads/' . $filename;

        if (file_exists($filepath)) {
            $this->response->download($filepath);
        } else {
            $this->response->set_status_code(404)
                           ->send_html('<h1>File not found</h1>', 404);
        }
    }
}

Form Submission with Redirect:

<?php
class ContactController extends Controller {

    public function submit() {
        if ( ! $this->request->is_post()) {
            $this->response->back();
            return;
        }

        $name = $this->request->post('name');
        $email = $this->request->post('email');
        $message = $this->request->post('message');

        if (empty($name) || empty($email) || empty($message)) {
            $this->session->set_flashdata('error', 'All fields are required');
            $this->response->back();
            return;
        }

        // Send email or save to database
        $this->email_model->send_contact($name, $email, $message);

        // Set success message and redirect
        $this->session->set_flashdata('success', 'Message sent successfully');
        $this->response->redirect('contact/thank-you');
    }

    public function thank_you() {
        $this->response->set_html_content($this->call->view('thank_you'))
                       ->send();
    }
}

RESTful API with Different Formats:

<?php
class ProductController extends Controller {

    public function index() {
        $products = $this->product_model->get_all();

        // Check if client wants JSON
        if ($this->request->wants_json()) {
            $this->response->send_json_success($products);
        } else {
            $this->response->set_html_content(
                $this->call->view('products', array('products' => $products), TRUE)
            )->send();
        }
    }

    public function show($id) {
        $product = $this->product_model->find($id);

        if ( ! $product) {
            if ($this->request->wants_json()) {
                $this->response->send_json_error('Product not found', 404);
            } else {
                $this->response->set_status_code(404)
                               ->set_html_content('<h1>404 - Product Not Found</h1>')
                               ->send();
            }
            return;
        }

        if ($this->request->wants_json()) {
            $this->response->send_json_success($product);
        } else {
            $this->response->set_html_content(
                $this->call->view('product', array('product' => $product), TRUE)
            )->send();
        }
    }
}

Best Practices

  • Always set appropriate status codes for responses (200 for success, 404 for not found, 500 for errors)

  • Use JSON helpers (send_json_success() and send_json_error()) for consistent API responses

  • Chain methods for cleaner and more readable code

  • Set secure cookie flags (secure, httponly, samesite) for sensitive data

  • Use ``send_no_content()`` for DELETE operations that return no data

  • Stream large files instead of loading them entirely into memory

  • Validate files exist before attempting to download

  • Use ``back()`` method for form submissions to improve user experience

  • Clear response when reusing the response object in complex operations

Security Considerations

Warning

Always validate and sanitize data before including it in responses, especially when outputting HTML or JSON that will be rendered by clients.

Security Features:

  • Automatic header injection prevention through proper header handling

  • Secure cookie options support (httponly, secure, samesite)

  • Status code validation for redirects (only allows safe codes)

  • File path validation in download method

Security Checklist:

<?php
// DO: Set secure cookies for sessions
$this->response->set_cookie('session', $token, 3600, '/', '', TRUE, TRUE, 'Strict');

// DO: Use appropriate status codes
$this->response->set_status_code(403)->send_json_error('Access denied');

// DO: Sanitize output
$name = htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8');
$this->response->set_html_content("<h1>$name</h1>");

// DON'T: Output raw user input
$this->response->set_html_content("<h1>{$_POST['name']}</h1>");

Troubleshooting

Common Issues and Solutions

Issue

Possible Solution

Headers already sent error

Ensure no output is sent before calling send() methods. Check for whitespace in PHP files

JSON encoding fails

Use json_last_error_msg() to debug. Ensure data contains UTF-8 encoded strings

File download not working

Verify file path is correct and file is readable. Check PHP memory limits for large files

Redirect not working

Ensure no output is sent before redirect. Check that headers haven’t been sent

Cookie not being set

Verify cookie name has no invalid characters. Check path and domain settings

404 pages not showing

Set status code before sending content. Use set_status_code(404) before send()

AJAX response not parsed

Ensure Content-Type header is set correctly (application/json for JSON responses)
