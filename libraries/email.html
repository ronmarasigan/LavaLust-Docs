Email Class

The Email class makes sending email simple and easy. It uses PHP’s built-in mail() function under the hood.

If you are planning to use more advanced features (SMTP, authentication, HTML templates, etc.), you may integrate third-party libraries such as PHPMailer or SwiftMailer instead.

Note

The Email class is designed for basic email functionality. For production applications with higher email volume or delivery requirements, consider using a dedicated email library or service.

Loading the Email Library

Before using the email features, load it via:

<?php
$this->call->library('email');

Basic Usage

Here is a basic example demonstrating how you might send an email. Note: This example assumes you are sending the email from one of your controllers.

Plain Text Email Example:

<?php
$this->call->library('email');

$this->email->sender('your@example.com', 'Your Name');
$this->email->recipient('someone@example.com');
$this->email->subject('Email Test');
$this->email->email_content('Testing the email class.');

if ($this->email->send()) {
    echo 'Email sent successfully!';
} else {
    echo 'Failed to send email.';
}

HTML Email Example:

<?php
$this->call->library('email');

$html_content = '<html>
    <body>
        <h1>Welcome!</h1>
        <p>Thank you for registering with our service.</p>
        <p><strong>Your account has been created successfully.</strong></p>
    </body>
</html>';

$this->email->sender('noreply@example.com', 'My Website');
$this->email->recipient('user@example.com');
$this->email->subject('Welcome to Our Website');
$this->email->email_content($html_content, 'html');
$this->email->send();

Email with Reply-To Address:

<?php
$this->call->library('email');

$this->email->sender('noreply@example.com', 'Support Team');
$this->email->reply_to('support@example.com'); // Replies go here
$this->email->recipient('customer@example.com');
$this->email->subject('Your Support Ticket');
$this->email->email_content('Your ticket has been received. We will respond within 24 hours.');
$this->email->send();

Sending to Multiple Recipients:

<?php
$this->call->library('email');

$recipients = array(
    'user1@example.com',
    'user2@example.com',
    'user3@example.com'
);

$this->email->sender('newsletter@example.com', 'Weekly Newsletter');
$this->email->recipient($recipients);
$this->email->subject('This Week\'s Updates');
$this->email->email_content('Here are the latest updates from our website...');
$this->email->send();

Sending Email with Attachment:

<?php
$this->call->library('email');

$this->email->sender('admin@example.com', 'Admin');
$this->email->recipient('client@example.com');
$this->email->subject('Your Invoice');
$this->email->email_content('Please find attached your invoice for this month.');
$this->email->attachment('/path/to/invoice.pdf');

if ($this->email->send()) {
    echo 'Invoice sent successfully!';
} else {
    echo 'Failed to send invoice.';
}

Complete Example with Error Handling:

<?php
class ContactController extends Controller {

    public function send_contact_form() {
        $this->call->library('email');

        $name = $this->input->post('name');
        $email = $this->input->post('email');
        $message = $this->input->post('message');

        // Prepare email
        $this->email->sender($email, $name);
        $this->email->reply_to($email);
        $this->email->recipient('contact@example.com');
        $this->email->subject('Contact Form Submission from ' . $name);
        $this->email->email_content("Name: $name\nEmail: $email\n\nMessage:\n$message");

        // Send and handle response
        if ($this->email->send()) {
            $this->session->set_flashdata('success', 'Thank you for contacting us!');
            redirect('contact/success');
        } else {
            $this->session->set_flashdata('error', 'Failed to send message. Please try again.');
            redirect('contact');
        }
    }
}

Method Chaining

All email preparation methods (except send()) support method chaining for more concise code:

<?php
$this->call->library('email');

$this->email->sender('noreply@example.com', 'Website')
            ->recipient('user@example.com')
            ->subject('Welcome!')
            ->email_content('Thank you for joining!')
            ->send();

Note

You must call $this->email->send() at the end to actually send the email. All other methods just prepare the email before sending.

Best Practices

  • Validate email addresses before sending to avoid delivery failures

  • Use descriptive subject lines that help recipients understand the email’s purpose

  • Always include a sender email address - many email servers reject emails without a valid sender

  • Set a reply-to address when the sender is a no-reply address so recipients can respond appropriately

  • Sanitize user input when using user-provided data in email headers to prevent header injection attacks

  • Test emails thoroughly before deploying to production

  • Consider rate limiting for contact forms to prevent spam

  • Use HTML emails carefully - ensure they have plain text alternatives for better compatibility

  • Log email sending attempts for debugging and audit purposes

Security Considerations

Warning

Never directly insert user input into email headers without proper validation and sanitization. This can lead to email header injection attacks.

Safe way to use user input:

<?php
// Validate email format
$user_email = filter_var($_POST['email'], FILTER_VALIDATE_EMAIL);

if ($user_email) {
    $this->email->recipient($user_email);
}

// Sanitize subject and content
$subject = htmlspecialchars($_POST['subject'], ENT_QUOTES, 'UTF-8');
$this->email->subject($subject);

Troubleshooting

Common Issues and Solutions

Issue

Possible Solution

Email not sending

Check that PHP’s mail() function is enabled on your server. Check server error logs.

Emails going to spam

Use proper email headers, avoid spam trigger words, consider using SMTP with authentication

Attachments not working

Verify the file path is correct and the file is readable by the web server

HTML emails displaying as raw code

Ensure you set the second parameter of email_content() to ‘html’

Multiple recipients not receiving email

Verify the email addresses are valid and properly formatted in the array

Headers injection warning

Always validate and sanitize user input before using in email headers

Limitations

  • Uses PHP’s built-in mail() function which may be limited by server configuration

  • Does not support SMTP authentication natively

  • No built-in queue system for handling large volumes of emails

  • No support for CC (Carbon Copy) or BCC (Blind Carbon Copy)

  • Limited attachment handling capabilities

For advanced email features, consider integrating:

  • PHPMailer - Full-featured email sending library with SMTP support

  • SwiftMailer - Powerful email library with many features

  • Transactional email services (SendGrid, Mailgun, Amazon SES) for production applications
