Response Class
==============

The **Response** class handles HTTP responses, status codes, headers, content types, and provides convenient methods for common response formats like JSON, HTML, XML, file downloads, and redirects.

Loading the Response Class
--------------------------

The Response class is automatically loaded by the LavaLust framework and is accessible via:

.. code-block:: php

    <?php
    $this->response;

No manual loading is required as it's available globally.

Properties
----------

.. list-table::
   :header-rows: 1

   * - Property
     - Type
     - Description
   * - ``$status_code``
     - int
     - HTTP status code (default: 200)
   * - ``$headers``
     - array
     - Response headers to be sent
   * - ``$content``
     - mixed
     - Response content body
   * - ``$cookies``
     - array
     - Cookies to set with the response
   * - ``$headers_sent``
     - boolean
     - Flag indicating if headers have already been sent

Methods
-------

Status Code Methods
~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``set_status_code($status_code)``
     - Sets the HTTP status code for the response.
   * - ``get_status_code()``
     - Returns the current HTTP status code.

Header Methods
~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``add_header($name, $value = '')``
     - Adds one or more HTTP headers to the response.
   * - ``remove_header($name)``
     - Removes a previously set header.
   * - ``get_headers()``
     - Returns all currently set headers.

Content Methods
~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``set_content($content)``
     - Sets the response content.
   * - ``get_content()``
     - Returns the current response content.
   * - ``append_content($content)``
     - Appends content to existing response content.
   * - ``set_html_content($content)``
     - Sets HTML content and adds Content-Type: text/html header.
   * - ``set_json_content($data, $options = 0)``
     - Sets JSON content and adds Content-Type: application/json header.

Response Sending Methods
~~~~~~~~~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``send()``
     - Sends the response including status code, headers, and content.
   * - ``send_json($data, $status_code = 200, $options = 0)``
     - Sends a JSON response immediately.
   * - ``send_json_success($data = NULL, $message = 'Success', $status_code = 200)``
     - Sends a standardized JSON success response.
   * - ``send_json_error($message, $status_code = 400, $additional_data = array())``
     - Sends a standardized JSON error response.
   * - ``send_html($content, $status_code = 200)``
     - Sends an HTML response immediately.
   * - ``send_text($content, $status_code = 200)``
     - Sends a plain text response.
   * - ``send_xml($xml, $status_code = 200)``
     - Sends an XML response.
   * - ``send_no_content()``
     - Sends a 204 No Content response (no response body).

Redirect Methods
~~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``redirect($url, $status_code = 302)``
     - Redirects the client to another URL.
   * - ``back($fallback = '/')``
     - Redirects back to the previous page.

File Methods
~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``download($filepath, $filename = NULL, $headers = array())``
     - Forces a file download.
   * - ``stream($callback, $headers = array())``
     - Streams content to the browser in chunks.

Cookie Methods
~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``set_cookie($name, $value = '', $expiration = 0, $path = '', $domain = '', $secure = FALSE, $httponly = FALSE, $samesite = 'Lax')``
     - Sets a cookie to be sent with the response.
   * - ``delete_cookie($name, $path = '', $domain = '')``
     - Deletes a cookie by setting its expiration to the past.

Utility Methods
~~~~~~~~~~~~~~~

.. list-table::
   :header-rows: 1

   * - Method
     - Description
   * - ``send_headers()``
     - Sends all queued headers and cookies without sending content.
   * - ``clear()``
     - Clears all response data (status code, headers, content, cookies).
   * - ``__toString()``
     - Magic method that returns response content as string.

Complete Examples
-----------------

**Basic JSON API Endpoint:**

.. code-block:: php

    <?php
    class ApiController extends Controller {
        
        public function users() {
            $users = $this->user_model->get_all();
            
            if ($users) {
                $this->response->send_json_success($users);
            } else {
                $this->response->send_json_error('No users found', 404);
            }
        }
        
        public function create_user() {
            $data = $this->request->post();
            
            if ($this->validate_user($data)) {
                $id = $this->user_model->create($data);
                $this->response->send_json_success(
                    array('id' => $id),
                    'User created successfully',
                    201
                );
            } else {
                $this->response->send_json_error('Invalid user data', 422);
            }
        }
    }

**File Upload with Response:**

.. code-block:: php

    <?php
    class FileController extends Controller {
        
        public function upload() {
            if ( ! $this->request->is_post()) {
                $this->response->set_status_code(405)
                               ->send_json_error('Method not allowed', 405);
                return;
            }
            
            $this->call->library('upload', $_FILES['file']);
            $this->upload->set_dir('uploads')->allowed_extensions(['jpg', 'png']);
            
            if ($this->upload->do_upload()) {
                $this->response->send_json_success(array(
                    'filename' => $this->upload->get_filename(),
                    'size' => $this->upload->get_size()
                ), 'File uploaded successfully');
            } else {
                $this->response->send_json_error($this->upload->get_errors(), 400);
            }
        }
        
        public function download($filename) {
            $filepath = 'uploads/' . $filename;
            
            if (file_exists($filepath)) {
                $this->response->download($filepath);
            } else {
                $this->response->set_status_code(404)
                               ->send_html('<h1>File not found</h1>', 404);
            }
        }
    }

**Form Submission with Redirect:**

.. code-block:: php

    <?php
    class ContactController extends Controller {
        
        public function submit() {
            if ( ! $this->request->is_post()) {
                $this->response->back();
                return;
            }
            
            $name = $this->request->post('name');
            $email = $this->request->post('email');
            $message = $this->request->post('message');
            
            if (empty($name) || empty($email) || empty($message)) {
                $this->session->set_flashdata('error', 'All fields are required');
                $this->response->back();
                return;
            }
            
            // Send email or save to database
            $this->email_model->send_contact($name, $email, $message);
            
            // Set success message and redirect
            $this->session->set_flashdata('success', 'Message sent successfully');
            $this->response->redirect('contact/thank-you');
        }
    }

**Method Chaining Example:**

.. code-block:: php

    <?php
    class UserController extends Controller {
        
        public function profile($id) {
            $user = $this->user_model->find($id);
            
            if ( ! $user) {
                $this->response->set_status_code(404)
                               ->add_header('X-Error-Source', 'UserController')
                               ->send_json_error('User not found', 404);
                return;
            }
            
            $this->response->set_status_code(200)
                           ->add_header('Cache-Control', 'private, max-age=300')
                           ->set_cookie('last_viewed_user', $id, 3600, '/')
                           ->send_json_success($user);
        }
    }

Best Practices
---------------

- **Always set appropriate status codes** for responses (200 for success, 404 for not found, 500 for errors)
- **Use JSON helpers** (``send_json_success()`` and ``send_json_error()``) for consistent API responses
- **Chain methods** for cleaner and more readable code
- **Set secure cookie flags** (``secure``, ``httponly``, ``samesite``) for sensitive data
- **Use ``send_no_content()``** for DELETE operations that return no data
- **Stream large files** instead of loading them entirely into memory
- **Validate files exist** before attempting to download
- **Use ``back()`` method** for form submissions to improve user experience

Security Considerations
------------------------

.. warning::

    Always validate and sanitize data before including it in responses, especially when outputting HTML or JSON that will be rendered by clients.

**Security Features:**

- Automatic header injection prevention through proper header handling
- Secure cookie options support (httponly, secure, samesite)
- Status code validation for redirects (only allows safe codes)
- File path validation in download method

Troubleshooting
-----------------

.. list-table:: Common Issues and Solutions
   :header-rows: 1

   * - Issue
     - Possible Solution
   * - Headers already sent error
     - Ensure no output is sent before calling send() methods
   * - JSON encoding fails
     - Use json_last_error_msg() to debug. Ensure data contains UTF-8 encoding
   * - File download not working
     - Verify file path is correct and file is readable
   * - Redirect not working
     - Ensure no output is sent before redirect
   * - Cookie not being set
     - Verify cookie name has no invalid characters. Check path and domain settings
   * - AJAX response not parsed
     - Ensure Content-Type header is set correctly (application/json for JSON responses)
