Class Overview

The Api class provides a full set of tools for building secure RESTful APIs in LavaLust Framework. It supports:

  • CORS handling
  • JSON body parsing
  • JWT access tokens
  • Refresh token management
  • Basic Authentication

Properties

refresh_token_table
string

Database table for storing refresh tokens.

payload_token_expiration
int

Access token validity in seconds (default: 900 = 15 min).

refresh_token_expiration
int

Refresh token validity in seconds (default: 604800 = 7 days).

allow_origin
string

Allowed CORS origin.

jwt_secret
string

Secret key for JWT signing.

Utility Methods

body()

Parses request body into an array (supports JSON, POST, raw input).

get_query_params()

Returns query parameters ($_GET).

require_method($method)

Ensures the request method matches, else sends 405 error.

respond($data, $code=200)

Send a JSON response with HTTP status code.

JWT Authentication

encode_jwt($payload)

Creates a signed JWT token.

decode_jwt($token)

Validates and decodes a JWT token.

require_jwt()

Requires a valid JWT or sends 401 Unauthorized.

Access & Refresh Tokens

issue_tokens($user_data)

Issues access and refresh tokens for a user.

{
  "access_token": "...",
  "refresh_token": "...",
  "expires_in": 900
}

refresh_access_token($refresh_token)

Generates new access token using refresh token.

revoke_refresh_token($refresh_token)

Revokes refresh token from DB.

Basic Authentication

require_basic_auth($user,$pass)

Forces Basic Auth, else 401 Unauthorized.

Example 1: Public Endpoint

class HelloController extends Controller {
    public function world() {
        $this->call->library('api');
        $api->respond(['message' => 'Hello World']);
    }
}

Example 2: Secure Endpoint (JWT)

class UserController extends Controller {
    public function profile() {
        $this->call->library('api');
        $payload = $api->require_jwt();
        $api->respond([
            'user_id' => $payload['sub'],
            'role' => $payload['role']
        ]);
    }
}

Example 3: Login & Issue Tokens

class AuthController extends Controller {
    public function login() {
        $this->call->library('api');
        $data = $api->body();
        if ($data['username']==='admin' && $data['password']==='secret') {
            $tokens = $api->issue_tokens(['id'=>1,'role'=>'admin']);
            $api->respond($tokens);
        }
        $api->respond_error('Invalid credentials',401);
    }
}

Example 4: Refresh Token

class AuthController extends Controller {
    public function refresh() {
        $this->call->library('api');
        $data = $api->body();
        if (!isset($data['refresh_token'])) {
            $api->respond_error('Missing refresh token',400);
        }
        $api->refresh_access_token($data['refresh_token']);
    }
}

Example 5: Basic Auth

class AdminController extends Controller {
    public function dashboard() {
        $api = new Api();
        $api->require_basic_auth('admin','1234');
        $api->respond(['message'=>'Welcome to admin dashboard']);
    }
}

Best Practices

Always store jwt_secret securely and rotate periodically.

Do not expose refresh tokens to client-side JS; keep them HttpOnly cookies.
