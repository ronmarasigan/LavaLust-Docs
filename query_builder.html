Class Overview

The Database class provides a fluent interface for working with databases in LavaLust PHP Framework. It supports:

  • Fluent query building
  • Prepared statements with parameter binding
  • Multiple database connections
  • Transactions with savepoints
  • Complex joins and conditions
  • Bulk insert/update operations
  • Pagination support

Note: All methods support method chaining for fluent query building.

Connection Methods

__construct($dbname = NULL)

Creates a new database connection instance.

Parameters

Parameter Type Description Default
$dbname string Database configuration name NULL (uses 'main' config)

Example

// Default connection
$this->call->database();

// Specific database configuration
$this->call->atabase('secondary_db');

instance($dbname)

Singleton pattern to get database instance.

Parameters

Parameter Type Description
$dbname string Database configuration name

Return Value

Database - Database instance

Example

$db = Database::instance('main');

CRUD Operations

table($table_name)

Sets the table to perform operations on.

Parameters

Parameter Type Description
$table_name string Table name (without prefix)

Return Value

$this - For method chaining

Example

$this->db->table('users')->get_all();

insert($fields)

Inserts a new record into the database.

Parameters

Parameter Type Description
$fields array Associative array of column => value

Return Value

integer - Last inserted ID

Example

$id = $this->db->table('users')->insert([
    'name' => 'John Doe',
    'email' => 'john@example.com'
]);

update($fields)

Updates records in the database.

Parameters

Parameter Type Description
$fields array Associative array of column => value

Return Value

integer - Number of affected rows

Example

$affected = $this->db->table('users')
    ->where('id', 1)
    ->update(['name' => 'Jane Doe']);

delete()

Deletes records from the database.

Return Value

integer - Number of affected rows

Example

$deleted = $this->db->table('users')
    ->where('id', 1)
    ->delete();

get($mode = PDO::FETCH_ASSOC)

Executes a SELECT query and returns a single row.

Parameters

Parameter Type Description Default
$mode integer PDO fetch mode PDO::FETCH_ASSOC

Return Value

array - Single row result

Example

$user = $this->db->table('users')->where('id', 1)->get();

get_all($mode = PDO::FETCH_ASSOC)

Executes a SELECT query and returns all matching rows.

Parameters

Parameter Type Description Default
$mode integer PDO fetch mode PDO::FETCH_ASSOC

Return Value

array - All matching rows

Example

$users = $this->db->table('users')->get_all();

Query Building

select($columns)

Specifies columns to select.

Parameters

Parameter Type Description
$columns string Comma-separated column names

Return Value

$this - For method chaining

Example

$this->db->table('users')->select('id, name, email')->get_all();

where() / or_where() / not_where() / or_not_where()

Adds WHERE conditions to the query.

Parameters (where)

Parameter Type Description Default
$where string|array Column name or array of conditions Required
$op string|array Operator or value if operator omitted NULL
$val mixed Value to compare NULL

Variations

  • or_where() - OR condition
  • not_where() - NOT condition
  • or_not_where() - OR NOT condition

Examples

// Simple where
$this->db->where('age', '>', 18)->get_all();

// Multiple conditions
$this->db->where([
    'status' => 'active',
    'age >' => 21
])->get_all();

// OR condition
$this->db->or_where('name', 'LIKE', '%john%')->get_all();

like() / or_like() / not_like() / or_not_like()

Adds LIKE conditions to the query.

Parameters (like)

Parameter Type Description Default
$field string Column name Required
$data mixed Value to search for Required

Examples

// Standard LIKE
$this->db->like('name', '%john%')->get_all();

// OR LIKE
$this->db->or_like('email', '%gmail%')->get_all();

// NOT LIKE
$this->db->not_like('status', 'inactive')->get_all();

in() / not_in() / or_in() / or_not_in()

Adds IN conditions to the query.

Parameters (in)

Parameter Type Description Default
$field string Column name Required
$keys array Array of values Required

Examples

// IN condition
$this->db->in('id', [1, 2, 3])->get_all();

// NOT IN
$this->db->not_in('status', ['deleted', 'banned'])->get_all();

// OR IN
$this->db->or_in('category_id', [5, 6])->get_all();

Join Methods

join($table_name, $cond, $type = '')

Adds a JOIN clause to the query.

Parameters

Parameter Type Description Default
$table_name string Table to join (without prefix) Required
$cond string Join condition Required
$type string Join type (INNER, LEFT, RIGHT, etc.) '' (INNER)

Example

$this->db->table('users')
    ->join('profiles', 'profiles.user_id = users.id')
    ->get_all();

inner_join() / left_join() / right_join() / full_outer_join()

Convenience methods for specific join types.

Examples

// INNER JOIN
$this->db->inner_join('orders', 'orders.user_id = users.id');

// LEFT JOIN
$this->db->left_join('comments', 'comments.post_id = posts.id');

// RIGHT JOIN
$this->db->right_join('departments', 'departments.id = employees.dept_id');

// FULL OUTER JOIN
$this->db->full_outer_join('archives', 'archives.item_id = items.id');

Aggregate Functions

select_max() / select_min() / select_sum() / select_count() / select_avg()

Adds aggregate functions to the SELECT clause.

Parameters (select_max)

Parameter Type Description Default
$column string Column to aggregate Required
$alias string Alias for the result NULL

Examples

// Maximum value
$this->db->select_max('age', 'max_age')->get();

// Row count
$this->db->select_count('id', 'total_users')->get();

// Average value
$this->db->select_avg('salary', 'average_salary')->get();

count()

Counts the number of rows in the table.

Return Value

integer - Number of rows

Example

$user_count = $this->db->table('users')->count();

Condition Methods

where_null() / where_not_null()

Adds IS NULL or IS NOT NULL conditions.

Parameters

Parameter Type Description
$where string Column name

Examples

// IS NULL
$this->db->where_null('deleted_at')->get_all();

// IS NOT NULL
$this->db->where_not_null('email_verified_at')->get_all();

between() / not_between() / or_between() / or_not_between()

Adds BETWEEN conditions to the query.

Parameters (between)

Parameter Type Description Default
$field string Column name Required
$value1 mixed Lower bound value Required
$value2 mixed Upper bound value Required

Examples

// BETWEEN
$this->db->between('age', 18, 30)->get_all();

// NOT BETWEEN
$this->db->not_between('salary', 30000, 50000)->get_all();

// OR BETWEEN
$this->db->or_between('experience', 5, 10)->get_all();

grouped(Closure $obj)

Groups conditions within parentheses.

Parameters

Parameter Type Description
$obj Closure Callback function with conditions

Example

$this->db->where('status', 'active')
    ->grouped(function($q) {
        $q->where('age', '>', 18)
          ->or_where('parent_consent', 1);
    })
    ->get_all();

Pagination & Limits

limit($limit, $end = NULL)

Adds a LIMIT clause to the query.

Parameters

Parameter Type Description Default
$limit integer Number of rows or start offset Required
$end integer End offset if two parameters used NULL

Examples

// Limit to 10 rows
$this->db->limit(10)->get_all();

// Limit with offset (20, 10)
$this->db->limit(20, 10)->get_all();

offset($offset)

Adds an OFFSET clause to the query.

Parameters

Parameter Type Description
$offset integer Offset value

Example

$this->db->limit(10)->offset(20)->get_all();

pagination($records_per_page, $page)

Sets up pagination for a query.

Parameters

Parameter Type Description
$records_per_page integer Number of records per page
$page integer Current page number

Example

$this->db->table('posts')
    ->pagination(10, 2) // Page 2 with 10 records per page
    ->get_all();

order_by($field_name, $order = null)

Adds an ORDER BY clause to the query.

Parameters

Parameter Type Description Default
$field_name string Column to order by Required
$order string Order direction (ASC/DESC) NULL (ASC)

Examples

// Order by name ascending
$this->db->order_by('name')->get_all();

// Order by date descending
$this->db->order_by('created_at', 'DESC')->get_all();

group_by($groupBy)

Adds a GROUP BY clause to the query.

Parameters

Parameter Type Description
$groupBy string|array Column(s) to group by

Example

$this->db->select('department, COUNT(*) as employee_count')
    ->group_by('department')
    ->get_all();

having($field, $op = null, $val = null)

Adds a HAVING clause to the query.

Parameters

Parameter Type Description Default
$field string Column name or condition Required
$op string|array Operator or value NULL
$val mixed Value to compare NULL

Example

$this->db->select('department, COUNT(*) as employee_count')
    ->group_by('department')
    ->having('employee_count', '>', 5)
    ->get_all();

Transactions

transaction()

Starts a database transaction.

Return Value

boolean - True on success

Example

$this->db->transaction();

commit()

Commits the current transaction.

Return Value

boolean - True on success

Example

$this->db->commit();

roll_back()

Rolls back the current transaction.

Return Value

boolean - True on success

Example

$this->db->roll_back();

Transaction Best Practices

  • Always pair transaction with either commit or rollback
  • Use try/catch blocks to handle transaction errors
  • Keep transactions as short as possible
  • Test transaction rollback scenarios

Utility Methods

last_id()

Gets the last inserted ID.

Return Value

integer - Last inserted ID

Example

$id = $this->db->last_id();

row_count()

Gets the number of rows affected by the last query.

Return Value

integer - Number of affected rows

Example

$count = $this->db->row_count();

get_sql()

Gets the last executed SQL query.

Return Value

string - The SQL query

Example

$sql = $this->db->get_sql();

raw($query, $args = array())

Executes a raw SQL query.

Parameters

Parameter Type Description Default
$query string SQL query to execute Required
$args array Parameters to bind []

Return Value

PDOStatement - Result statement

Example

$result = $this->db->raw("SELECT * FROM users WHERE age > ?", [18]);

Best Practices

  • Always use parameter binding to prevent SQL injection
  • Use transactions for operations that require multiple queries
  • Specify only needed columns with select()
  • Use pagination for large result sets
  • Close connections when done (handled automatically in destructor)
